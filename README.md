# AI Constructor - IT Project Platform

Полнофункциональная платформа для создания, управления и версионирования IT-проектов с поддержкой CRUDL операций.

## Функциональность

### 1. Управление проектами (CRUDL)

#### **Create** - Создание проектов
- Создавайте новые проекты через модальную форму
- Указывайте тип проекта (Web App, Mobile, API, Full Stack)
- Выбирайте фреймворки (React, Vue, Express.js, FastAPI и т.д.)
- Устанавливайте приоритет и сроки

#### **Read** - Просмотр деталей
- Откройте проект для просмотра всех его деталей
- Просмотрите список файлов
- Изучите историю версий
- Проверьте информацию о технологиях

#### **Update** - Редактирование проектов
- Редактируйте информацию о проекте (название, описание, статус)
- Обновляйте выбранные фреймворки и технологии
- Изменяйте приоритет и сроки
- Редактируйте содержимое файлов проекта

#### **Delete** - Удаление
- Удаляйте проекты с подтверждением
- Удаляйте файлы из проектов
- Все связанные данные удаляются автоматически

#### **List** - Список проектов
- Просмотрите все проекты на главной странице
- Фильтруйте по статусу (Planning, In Progress, Completed, On Hold)
- Ищите проекты по названию и описанию
- Группировка по типу и технологиям

### 2. Управление файлами

- **Создание файлов**: Добавляйте новые файлы в проект с выбором языка программирования
- **Редактирование кода**: Встроенный редактор кода с синтаксисом (Dark theme)
- **Копирование**: Быстро копируйте код в буфер обмена
- **Скачивание**: Загружайте файлы на компьютер
- **Удаление**: Удаляйте ненужные файлы

### 3. История версий

- **Создание версий**: Сохраняйте снимки проекта с описанием изменений
- **Просмотр истории**: Смотрите все версии с датой создания
- **Отслеживание изменений**: Каждая версия содержит информацию о том, когда и что было изменено

## Архитектура базы данных

### Таблица: `projects`
Основная информация о проектах:
- `id` - Уникальный идентификатор
- `title` - Название проекта
- `description` - Описание
- `status` - Статус (planning, in_progress, completed, on_hold)
- `priority` - Приоритет (low, medium, high)
- `project_type` - Тип (web, mobile, api, fullstack)
- `backend_framework` - Backend фреймворк
- `frontend_framework` - Frontend фреймворк
- `database_type` - Тип БД (postgresql, mongodb, sqlite, mysql)
- `start_date`, `end_date` - Даты проекта
- `created_at`, `updated_at` - Временные метки

### Таблица: `project_files`
Файлы внутри проектов:
- `id` - Уникальный идентификатор
- `project_id` - Ссылка на проект
- `name` - Имя файла
- `path` - Путь файла
- `content` - Содержимое файла
- `language` - Язык программирования
- `created_at`, `updated_at` - Временные метки

### Таблица: `project_versions`
История версий проектов:
- `id` - Уникальный идентификатор
- `project_id` - Ссылка на проект
- `version_number` - Номер версии
- `message` - Описание изменений
- `created_at` - Дата создания
- `created_by` - Кто создал версию

## Компоненты

### `ProjectCard` (`src/components/ProjectCard.tsx`)
Отображает карточку проекта на главной странице:
- Название и тип проекта
- Описание
- Выбранные фреймворки и БД
- Статус и приоритет
- Действия: открыть, редактировать, удалить

### `ProjectForm` (`src/components/ProjectForm.tsx`)
Форма для создания/редактирования проектов:
- Основная информация (название, описание)
- Выбор типа и статуса
- Выбор технологий (frontend, backend, БД)
- Даты начала и завершения

### `ProjectDetails` (`src/components/ProjectDetails.tsx`)
Полный просмотр проекта:
- Левая панель со списком файлов
- Центральная область для редактирования кода
- Управление файлами
- Просмотр истории версий

### `CodeEditor` (`src/components/CodeEditor.tsx`)
Встроенный редактор кода:
- Синтаксисная подсветка
- Горячие клавиши для сохранения
- Копирование и скачивание файлов
- Создание версий с описанием изменений

### `VersionHistory` (`src/components/VersionHistory.tsx`)
Управление историей версий:
- Просмотр всех версий
- Информация о времени создания
- Создание новой версии с описанием

## Как начать работу

1. Создайте новый проект через кнопку "New Project"
2. Заполните основную информацию и выберите технологии
3. Откройте проект нажатием зеленой кнопки
4. Добавьте файлы через кнопку "New File"
5. Редактируйте код в встроенном редакторе
6. Сохраняйте изменения и создавайте версии
